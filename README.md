# 不全书

`不全书（BuQuanShu / Introduction to Classical Music）` 是一个面向 `Windows` 的本地古典音乐资料库项目。它把资料维护、静态站点构建和桌面入口整合在一起，适合希望在本机长期整理作曲家、人物、团体、作品、版本与专题专栏的音乐爱好者。

你不需要会写代码，也可以直接使用安装版开始维护自己的古典音乐资料库。

这个仓库公开发布的是：

- 源代码仓库
- Windows 安装版：`不全书 Setup <版本号>.exe`
- Windows 便携版：后续发布

默认发布库是一个干净的空库模板，只附带《不全书使用手册》专栏和手册图片，不包含任何测试条目或开发者个人资料。

## 这个项目可以做什么

- 维护自己的古典音乐资料库
- 创建作曲家、人物、团体、作品、版本和专栏
- 运行自动检查与候选审查，辅助补全条目信息
- 把资料库构建成可浏览的本地静态站点
- 用桌面启动器统一打开站点、维护工具和版本检索工具
- 给版本条目添加外部资源链接和本地资源链接

## 适合谁

- 想长期整理自己收听目录的古典音乐爱好者
- 希望把私人资料库做成可浏览站点的个人用户
- 需要在本地维护结构化音乐资料的研究、写作或社群组织者
- 想在此基础上继续开发的协作者

## 如何安装

### 方式一：下载安装版

这是最适合普通用户的方式，也是后续公开发布时的主要推荐方式。

1. 前往项目 `Releases` 页面。
2. 下载 `不全书 Setup <版本号>.exe`。
3. 运行安装程序，按提示完成安装。
4. 安装完成后，启动 `不全书`。

安装版特点：

- 支持自定义安装路径
- 安装后自带默认资料库 `library`
- 默认资料库位于程序安装目录中
- 升级时可直接覆盖旧版本，不要求先卸载

卸载说明：

- 卸载会删除程序目录和其中自带的默认资料库
- `%APPDATA%\Introduction to Classical Music` 下的运行时日志和设置默认保留

### 方式二：从源代码安装

适合希望自己构建、调试或参与开发的用户。普通用户通常不需要选择这一方式。

准备环境：

- `Windows 10/11 x64`
- `Node.js 22`
- `npm`
- `Python 3.13`

执行步骤：

```powershell
git clone <仓库地址>
cd introduction-to-classical-music-public
npm run bootstrap:windows
npm run doctor:windows
npm run check
npm run build
```

如需生成安装版：

```powershell
npm run package:windows
```

### 方式三：下载便携版

便携版将作为后续公开发布资源提供，适合：

- 免安装试用
- 演示和携带
- 不想改动系统安装状态的使用场景

当前仓库已经为便携版保留构建链，但首个公开版本仍以安装版为主。

## 第一次打开后怎么用

启动器会提供三个入口：

- **打开本地构建库**：打开当前资料库已经构建好的站点
- **打开维护工具**：创建、修改和检查条目
- **打开版本检索工具**：辅助整理版本条目候选信息

推荐的使用顺序：

1. 进入维护工具。
2. 创建作曲家、人物、团体和作品等基础条目。
3. 再创建版本条目，并补充图片、链接和导览说明。
4. 运行自动检查或候选审查。
5. 点击“构建不全书”。
6. 用“打开本地构建库”检查前台效果。
7. 需要备份或迁移时导出整个资料库。

## 默认资料库说明

公开发布的默认资料库遵循以下边界：

- `composer / person / work / recording` 默认为空
- 只保留《不全书使用手册》专栏
- 手册图片来自 `materials/default-library/usage-guide`
- 不包含任何测试条目
- 不包含开发者个人资料库

本地资源链接说明：

- 维护工具允许给版本条目添加本地资源链接
- 这些链接会显示在站点前台
- 但它们通常只在当前设备有效
- 如果把资料库导出到其他设备，本地链接一般需要重新整理

## 仓库大致结构

```text
.
|-- apps/
|   |-- desktop/   # 桌面启动器
|   |-- owner/     # 维护工具
|   `-- site/      # 静态站点
|-- data/          # 默认公开种子数据
|-- docs/          # 发布与协作文档
|-- materials/     # 默认手册图片等公开素材
|-- packages/      # 共享核心逻辑
|-- scripts/       # 构建和检查脚本
|-- tests/         # 自动化测试
`-- tools/recording-retrieval-service/app/
    `-- ...        # 版本检索服务源码
```
