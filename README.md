# 不全书

**不全书（BuQuanShu / Introduction to Classical Music）** 是一个面向 `Windows` 的本地古典音乐资料库应用。  
它把“资料整理”“站点构建”“桌面启动”三件事放进同一套工具里，适合希望长期维护自己古典音乐目录的普通乐迷、研究者和社群组织者。

你可以把它理解成一套专门为古典音乐整理设计的个人资料库：

- 先在维护工具里录入作曲家、人物、团体、作品和版本
- 再一键构建出一套可以浏览的本地站点
- 以后继续补充、修订、导出和迁移这份资料库

默认安装后会附带一个**干净的空资料库模板**，只包含《不全书使用手册》专栏，不包含任何测试条目或开发者个人资料。

## 它适合谁

- 想长期整理自己收听过的古典音乐版本
- 想把私人目录做成可浏览的本地站点
- 需要把作曲家、作品、人物、团体和版本建立成结构化资料
- 想自己维护一份可持续更新的音乐导览库

## 它能做什么

- 创建和维护作曲家、人物、团体、作品、版本、专栏
- 为版本条目添加图片、导览文字、外部链接和本地资源链接
- 运行自动检查（auto check，自动检查）和候选审查（review，候选审查）
- 将资料库构建为静态站点（static site，静态站点）
- 通过桌面启动器统一打开站点、维护工具和版本检索工具
- 导入、导出、备份和迁移自己的资料库

## 安装方式

### 1. 安装版 exe

这是最适合普通用户的方式。

1. 前往本项目的 [Releases](https://github.com/Hektor1277/introduction-to-classical-music/releases) 页面。
2. 下载 `不全书 Setup <版本号>.exe`。
3. 双击运行安装程序。
4. 按提示选择安装目录并完成安装。
5. 安装完成后，从开始菜单或桌面启动 `不全书`。

安装版特点：

- 支持自定义安装路径
- 安装完成后，程序目录中自带一个 `library` 默认资料库
- 升级时可以直接覆盖旧版本，不要求先卸载
- 默认资料库是空模板，只附带《不全书使用手册》

卸载说明：

- 卸载会删除程序目录以及程序目录中的内置默认资料库
- `%APPDATA%\Introduction to Classical Music` 下的日志与运行设置默认保留

### 2. 从源码安装

适合希望自己构建、调试或参与后续开发的用户。

准备环境：

- `Windows 10/11 x64`
- `Node.js 22`
- `npm`
- `Python 3.13`

执行步骤：

```powershell
git clone https://github.com/Hektor1277/introduction-to-classical-music.git
cd introduction-to-classical-music
npm run bootstrap:windows
npm run doctor:windows
npm run check
npm run build
```

如果你还希望自己打包安装版：

```powershell
npm run package:windows
```

### 3. 便携版

便携版（portable edition，便携版）会在后续公开发布。  
它适合临时试用、演示、随身携带，或者不想改动系统安装状态的使用场景。

当前仓库已经为便携版保留了构建链路，但首个公开发布版本以安装版为主。

## 第一次打开后怎么用

启动器里会看到三个入口：

- **打开本地构建库**：打开当前资料库已经构建好的站点
- **打开维护工具**：录入、修改、检查和构建资料库
- **打开版本检索工具**：辅助处理版本条目的候选信息

推荐的第一次使用顺序：

1. 打开维护工具。
2. 先创建基础条目：作曲家、人物、团体、作品。
3. 再创建版本条目，补充图片、导览文字和资源链接。
4. 运行自动检查与候选审查。
5. 点击“构建不全书”。
6. 用“打开本地构建库”检查前台站点效果。
7. 需要备份时导出整个资料库。

## 默认资料库包含什么

公开发布的默认资料库遵循以下边界：

- `composer / person / work / recording` 默认全部为空
- 只保留《不全书使用手册》专栏
- 手册图片来自 `materials/default-library/usage-guide`
- 不包含任何测试条目
- 不包含开发者个人资料库

也就是说，安装后你拿到的是一个**干净的起点**，而不是一个已经预填了演示内容的样板库。

## 关于资源链接

维护工具允许给版本条目添加两类资源链接：

- 外部资源链接：例如网站、流媒体页面、视频页面
- 本地资源链接（local link，本地链接）：例如你本机上的音频、视频或文档

本地资源链接会显示在前台站点中，但通常只在当前设备有效。  
如果你把资料库导出到另一台设备，这类链接一般需要重新整理。

## 当前公开发布包含什么

当前公开发布的资源包括：

- GitHub 源代码仓库
- Windows 安装版 `不全书 Setup <版本号>.exe`

后续计划补充：

- Windows 便携版
- 更细化的升级、迁移与分发说明

## 项目边界

当前公开版本有以下边界：

- 目前只正式支持 `Windows 10/11 x64`
- 首发版本不提供应用内自动更新
- 默认资料库是空模板，不是演示资料库
- 本地资源链接只在当前设备有效
