# 项目边界说明

为了避免安装版、便携版和互联网部署站点互相串扰，当前发布体系固定分为三个边界清晰的部分。

## 1. 安装版源码仓库

路径：

- 当前公共源码仓库

职责：

- 维护桌面应用源码
- 维护维护工具、站点构建工具和版本检索工具源码
- 生成默认空资料库
- 生成 Windows 安装版

发布约束：

- 仓库必须保持为干净源码状态
- 默认资料库必须是空库，只附带《不全书使用手册》
- 不保存个人资料库内容
- 不保存便携版站点归档

## 2. 便携版发布资产

路径：

- 例如：`<private-assets-root>\releases`

职责：

- 保存便携版 `EXE` 与 `ZIP`
- 保存安装版安装程序 `EXE`

发布约束：

- 这里只保存二进制产物，不建立 Git 仓库
- 便携版应基于个人资料库站点构建
- 便携版运行时不依赖构建机器上的个人资料库路径
- 安装版与便携版的输出目录必须分离，不能相互覆盖

## 3. 互联网部署仓库

路径：

- 独立的互联网部署仓库

职责：

- 保存准备上线的静态站点版本
- 保存部署脚本、审计脚本和阿里云配置模板

发布约束：

- 不包含桌面应用源码
- 不包含维护工具源码
- 不包含检索服务源码
- 不包含安装版或便携版二进制文件

## 4. 个人资料库与站点源

路径：

- 例如：`<private-assets-root>\libraries\不全书-个人资料库`
- 例如：`<private-assets-root>\sites`

职责：

- 作为个人内容的唯一真实源头
- 便携版和互联网部署站点都从这里复制构建结果

发布约束：

- 其他工作区只能做副本或构建缓存
- 不反向把构建目录当成源头继续编辑

## 5. 强制隔离规则

- 安装版输出目录：`output/releases`
- 便携版输出目录：`output/releases-portable`
- 便携版构建站点暂存目录：`output/portable-site-staging`
- 互联网部署仓库只从站点归档目录同步，不从源码仓库直接抓取构建缓存

只要继续遵守这几个边界，三个项目就不会再互相污染。
